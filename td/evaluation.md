## Évaluation des TDs

Chaque TD comprend **2 à 3 séances de 2h** et est noté sur 20.

### Critères d'évaluation

Chaque TD a **ses propres critères d'évaluation** en fonction des objectifs pédagogiques. Consultez le fichier spécifique de chaque TD pour connaître ses objectifs.


**Éléments communs à tous les TDs** :
- ✅ Respect de l'architecture hexagonale
- ✅ Qualité du code et tests
- ✅ **Engagement en TD** : Votre présence et participation aux séances sont valorisées

### Comment valider un TD

1. **Travaillez progressivement** pendant les séances de 2h
2. **Commitez régulièrement** votre avancement
3. **Poussez un tag final** à la fin du TD complet :
   ```bash
   git tag TD1
   git push origin TD1
   ```

**Tags intermédiaires optionnels** : Vous pouvez pousser des tags après chaque séance (TD1a, TD1b...) pour recevoir un feedback automatique sans note définitive.

## Utilisation des assistants IA

### En dehors des séances de TD
✅ **Autorisée** : Vous pouvez utiliser des assistants IA (ChatGPT, GitHub Copilot, etc.) pour :
- Générer du code
- Comprendre des concepts
- Débugger

⚠️ **Important** : Vous devez comprendre le code généré par l'IA. Le QCM final évalue votre compréhension personnelle.

### Pendant le QCM final
**Interdite** : l'utilisation de l'IA pendant le QCM en présentiel sera assimilée à une tricherie, et sanctionnée comme telle.

---

## Conseils pour réussir

1. **Commitez souvent** pendant les TD (toutes les 15-20 minutes)
2. **Testez votre code** avec \`pytest\` avant chaque commit
3. **Lisez les feedbacks** automatiques après chaque tag
4. **Améliorez progressivement** votre code entre les séances
5. **Venez en TD** : c'est l'occasion d'avoir de l'aide et de progresser
